# Card

The Card component is a flexible container that provides consistent styling and elevation for content grouping.

## Features

- **Variants**: Default, Elevated, and Outlined styles
- **Flexible Padding**: Configurable padding using design tokens
- **Consistent Styling**: Rounded corners and proper spacing
- **Shadow Support**: Elevation shadows for visual hierarchy
- **Customizable**: Support for custom styles and content

## Design Tokens

The Card component uses the following design tokens:

- **Colors**: `colors.surface`, `colors.border`
- **Spacing**: `spacing.*` for padding options
- **Radius**: `radius.lg` for rounded corners
- **Shadows**: `shadows.md` for elevation

## Accessibility

- Semantic container structure
- Proper contrast ratios
- Screen reader compatible
- Keyboard navigation support for interactive content

## Usage Guidelines

### Do
- Use cards to group related content together
- Maintain consistent padding within similar card types
- Use elevation to show hierarchy and importance
- Keep card content focused and scannable
- Use appropriate variants for different contexts

### Don't
- Don't overcrowd cards with too much content
- Don't use cards for single pieces of information
- Don't nest cards too deeply
- Don't use inconsistent spacing within cards
- Don't make cards too narrow or too wide

## Layout Patterns

### Grid Layout
Use cards in grid layouts for consistent content presentation:

```jsx
<View style={{ flexDirection: 'row', flexWrap: 'wrap', gap: 16 }}>
  <Card style={{ flex: 1, minWidth: 200 }}>
    {/* Card content */}
  </Card>
  <Card style={{ flex: 1, minWidth: 200 }}>
    {/* Card content */}
  </Card>
</View>
```

### List Layout
Use cards in vertical lists for detailed content:

```jsx
<View style={{ gap: 16 }}>
  <Card variant="outlined">
    {/* List item content */}
  </Card>
  <Card variant="outlined">
    {/* List item content */}
  </Card>
</View>
```</Canvas>
