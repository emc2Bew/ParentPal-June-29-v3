# Button

The Button component is a versatile, accessible button that supports multiple variants, sizes, and states.

## Features

- **Variants**: Primary, Secondary, and Outline styles
- **Sizes**: Small, Medium, and Large
- **States**: Normal, Disabled, and Loading
- **Accessibility**: Full screen reader support with proper ARIA attributes
- **Touch Target**: Minimum 44pt touch target for accessibility compliance

## Design Tokens

The Button component uses the following design tokens:

- **Colors**: `colors.primary`, `colors.accent`, `colors.textInverse`
- **Typography**: `typography.fontWeight.semibold`, `typography.fontSize.*`
- **Spacing**: `spacing.md`, `spacing.lg`, `spacing.xl`
- **Radius**: `radius.md`
- **Dimensions**: `dimensions.touchTarget`, `dimensions.buttonHeight.*`

## Accessibility

- Minimum 44pt touch target
- Proper `accessibilityRole="button"`
- Support for `accessibilityLabel` and `accessibilityState`
- Screen reader compatible
- Keyboard navigation support (web)

## Usage Guidelines

### Do
- Use Primary buttons for the main action on a screen
- Use Secondary buttons for secondary actions
- Use Outline buttons for tertiary actions or in button groups
- Provide clear, action-oriented button text
- Use loading state for async operations

### Don't
- Don't use more than one Primary button per screen section
- Don't make button text too long (keep under 3 words when possible)
- Don't disable buttons without providing feedback about why
- Don't use buttons for navigation (use links instead)</Canvas>
